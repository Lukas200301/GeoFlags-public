# Frontend Setup Instructions

Complete guide to set up and run the GeoFlags frontend application.

## Prerequisites

Before you begin, ensure you have:

- **Node.js 24.11.1+** ([Download](https://nodejs.org/))
- **npm** or **yarn** package manager
- **Git** version control
- **Backend API** running (see [backend setup](backend/SETUP_INSTRUCTIONS.md))

## Quick Start

### 1. Install Dependencies

```bash
npm install
```

This installs all required packages:

- Nuxt 4
- TypeScript
- TailwindCSS v4
- Socket.io client
- Vue 3 and ecosystem

### 2. Configure Environment

Create your environment file:

```bash
cp .env.example .env
```

Edit `.env` with your backend URLs:

```env
NUXT_PUBLIC_API_BASE=http://localhost:3001
NUXT_PUBLIC_WS_URL=http://localhost:3001
```

> Replace `localhost:3001` with your actual backend URL if different.

### 3. Verify Setup

Run the verification script to check everything is configured correctly:

```bash
node verify-setup.js
```

This validates:

- ✅ Node.js version compatibility
- ✅ Dependencies installed
- ✅ Environment configuration
- ✅ Required directories exist

### 4. Start Development Server

```bash
npm run dev
```

The application will be available at **http://localhost:3000**

## Security Features

The frontend implements enterprise-grade security:

- **HttpOnly Cookie Authentication** - Tokens never exposed to JavaScript
- **CSRF Protection** - All mutations require CSRF tokens
- **Role-based Access Control** - Admin routes protected by middleware
- **Server-side Verification** - All permissions verified by backend
- **Type Safety** - TypeScript strict mode enabled

> See [SECURITY.md](SECURITY.md) for detailed security documentation.

## Troubleshooting

### Cannot Connect to Backend

**Symptoms**: API requests fail, cannot login/register

**Solutions**:

1. Verify backend is running at URL in `.env`
2. Check CORS settings on backend allow credentials
3. Restart dev server after changing `.env`
4. Check browser console for detailed errors

### CSRF Token Errors

**Symptoms**: Requests blocked with CSRF validation errors

**Solutions**:

1. Ensure backend implements `/api/auth/csrf-token` endpoint
2. Verify CSRF token validation in backend
3. Clear browser cookies and retry
4. Confirm `useApi()` composable is used for API calls

### Admin Access Denied

**Symptoms**: Cannot access `/admin` routes

**Solutions**:

1. Verify user has admin role in backend database
2. Log out and log back in to refresh session
3. Check backend `/api/auth/me` returns `role: 'admin'`
4. Verify admin middleware is functioning

### Socket.io Connection Failed

**Symptoms**: Real-time updates not working

**Solutions**:

1. Check `NUXT_PUBLIC_WS_URL` in `.env`
2. Verify backend Socket.io server is running
3. Confirm CORS allows WebSocket credentials
4. Check browser console for connection errors

### TypeScript Errors in IDE

**Symptoms**: Red squiggles in code editor

**Solutions**:

1. Restart TypeScript server in VS Code
2. Run `npm run dev` to generate `.nuxt` types
3. Ensure `node_modules` is installed
4. Verify `tsconfig.json` extends `.nuxt/tsconfig.json`

## Development Best Practices

### Code Organization

- Keep components small and focused (single responsibility)
- Use composables for reusable logic
- Store types in `types/index.ts`
- Follow existing naming conventions

### TypeScript

- Enable strict mode for type safety
- Define interfaces for all data structures
- Avoid `any` type - use proper types
- Leverage IDE autocomplete

### Security

- **Never** store tokens in localStorage
- **Always** use CSRF tokens for mutations
- **Verify** roles server-side (client checks are UX only)
- **Validate** user input on both client and server
- **Use** HTTPS in production

### Performance

- Use `v-if` for conditional rendering
- Lazy load heavy components with `defineAsyncComponent`
- Use `v-once` for static content
- Implement virtual scrolling for long lists
- Optimize and compress images
